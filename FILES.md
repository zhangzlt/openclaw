# FILES.md - 工作空间文件职责

> 一个信息只有一个源头。其他文件只引用，不复制。

## 文件职责速查

| 文件 | 职责 | 内容 |
|------|------|------|
| AGENTS.md | 行为规则 | 权限、红线、安全分级、开发流程、沟通规则 |
| SOUL.md | 性格设定 | 说话风格、语气、核心原则、团队关系 |
| IDENTITY.md | 身份定义 | 名字、角色、定位、能力范围、技术栈 |
| USER.md | 导师信息 | 导师身份、协作规则 |
| HEARTBEAT.md | 心跳清单 | 定时检查项 |
| WORKFLOWS.md | 工作流程 | 项目SOP、沟通流程 |
| TOOLS.md | 工具速查 | 环境地址、命令模板、账号（无密码） |
| MEMORY.md | 长期记忆 | 决策、经验教训、团队关系精华 |
| FILES.md | 本文档 | 文件职责索引 |
| members/ | 画像文件 | 每个团队成员的详细画像 |
| memory/ | 每日日志 | 每天 `YYYY-MM-DD.md` |
| secrets/ | 凭证 | 密码、token（不在 git 中） |
| work/ | 项目目录 | 全部项目代码和产出 |

## 引用规则

- 要密码 → `见 secrets/xxx`
- 要系统地址 → 查 TOOLS.md
- 要怎么做事 → 查 WORKFLOWS.md
- 要权限规则 → 查 AGENTS.md
- 要某人偏好 → 查 `members/<user_id>.md`
- 要代码仓库/分支 → 查 TOOLS.md 或对应 skill

## 文件管理规则

- MEMORY.md 保持 <5KB，超限按优先级精简
- 每个日志文件保持 <5KB
- TOOLS.md 保持 <10KB
- 截图放 `screenshots/<任务名>/`，不堆根目录
- 临时探索文件任务完成后立即删除
- 每个信息只存在一个文件中
