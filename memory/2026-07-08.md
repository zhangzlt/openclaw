### URL 字段分析
`inspect_daily.py` 中从 API 获取的 `url` 字段格式不统一：
- `feishuapp.cn/ai/gui/chat/a_xxx`（真正的聊天页）：仅 6/31
- `aily.feishu.cn/agents/agent_xxx`（Agent 配置页）：13/31
- `aiforce.cloud/app/app_xxx`（应用页）：5/31
- 其他域名 + 无 URL：7/31

结论：不能靠 `url` 字段直接拼聊天页 URL。

### 核心阻塞：飞书 SSO 跨域登录
- `agent.digitalchina.com` 的 localStorage 登录态不带给内嵌的 `feishu.cn` iframe
- 两种 `openType` 都受影响：
  - `embed`（8个）：内嵌模式，iframe 里是飞书登录页
  - `newTab`（22个）：新标签页跳转到飞书
- API 直接对话也失败（返回"系统繁忙"）

### 待决策

1. Playwright 飞书 SSO 登录：在浏览器中完成一次飞书账号密码登录，之后所有飞书系 iframe/标签页共享登录态
