## Agent Market 巡检报告飞书文档截图定位问题（2026-07-10 上午）

### 问题
飞书文档中截图全部堆在文档末尾，没有插入到对应智能体的"截图："和"用时："之间。

### 根因分析
1. `feishu_doc write` 的 `![](url)` 自动上传功能对 `http://127.0.0.1:18990/...` localhost URL 不生效（`images_processed: 0`）
2. `feishu_doc upload_image` 使用 `block_id` + `index` 参数时，`block_id` 被当作 `parent_block_id`（父级 block），而非"插入到该 block 之后"
3. 之前生成的 markdown 中图片 URL 丢失了 `screenshots/` 路径前缀

### 验证可行的方案
**交替 append → upload_image → append** 模式：
1. `feishu_doc append` 追加智能体文字内容（含"截图："行）
2. `feishu_doc upload_image`（不传 block_id，不传 index）→ 天然追加到文档末尾，紧随文字
3. `feishu_doc append` 追加"用时：..."→ 在图片之后
4. 循环下一个智能体

此模式下 upload_image 不传 block_id 会自动追加到末尾，配合 append 形成正确的文字-图片-文字顺序。
