### feishu_build_doc_v2.py：跨进程逐 section 交错方案（15:00-15:08）

**核心发现**：`add_blocks` 传 `block_type=27`（image）始终返回 `1770001 invalid param`，但 `feishu_doc upload_image` 工具可以创建 image block。因此无法在 Python 脚本内一次 API 调用完成"文字+图片"。

**最终方案**：
1. `feishu_build_doc_v2.py` 不再操作飞书 API，只做指令生成（门禁校验 → 输出 `INSTRUCTIONS.json`）
2. Cron agent 读取 `INSTRUCTIONS.json`，逐 agent 调用：
   - `feishu_doc append(doc_token, text_markdown)` → 写文字
   - `feishu_doc upload_image(doc_token, file_path)` → 插截图
3. 因为每次 append 都追加到文档末尾，交错顺序天然正确：Agent1 文字 → Agent1 截图 → Agent2 文字 → Agent2 截图 → ...

**验证结果**（文档 `LAl9dDOkRomyNMxYZ4eclZLBnyc`）：
- block 顺序：Heading3(126) → Text × N → **Image(126)** → Heading3(125) → Text × N → **Image(125)** ✅
- 截图紧贴对应 agent，不堆末尾
