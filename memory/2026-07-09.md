## Cron 超时问题与优化方案讨论

### 问题
- Cron job 连续 4 次超时失败，每次 ~900s（1200s 上限）
- 根因：cron agent 做飞书投递需 25+ 轮对话（创建文档 + 写内容 + 逐张上传 20 张截图），deepseek-v4-pro 每次思考 15-30s，投递环节吃掉 8-12 分钟
- 脚本本身约 7 分钟，加上投递总耗时超限

### 三套优化方案

**方案 A：批量并行上传（最小改动，推荐先试）**
- cron agent 一轮同时调多个 `upload_image`，4 轮搞定 20 张
- 模型轮次从 25→8，节省 ~8 分钟
- 改动仅 cron job prompt

**方案 B：Python 直驱飞书 API（最彻底）**
- 文档创建 + 内容写入 + 截图上传全在 `inspect_daily.py` 里完成
- 直接用飞书 HTTP API，不需要 agent 逐轮对话
- 总耗时 ~8 分钟，彻底解决超时

**方案 C：Canvas HTML 报告（绕开飞书限制）**
- Python 生成带内嵌截图的 HTML，托管 OpenClaw Canvas
- cron agent 只发 Canvas 链接
- 看报告体验好，但不在飞书里

### 决策
- 导师明确：**绝对不能容忍截图堆末尾**
- 待导师从 A/B/C 中选择或组合
