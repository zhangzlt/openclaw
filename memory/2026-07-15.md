## 飞书原生代码块支持（block_type=14）

### 背景
之前用 text_block 包裹 Markdown 代码围栏（```...```），飞书不解 Markdown，显示为普通文字 + 反引号。
且回答被拆成单字换行（"\n".join(string) 对字符串逐字分割的 bug）。

### 解决方案
- **native_code_block()**: 使用 block_type=14，language=1 (Plain Text)
- **关键发现**: 飞书 API 会截断 text_run.content 中第一个 `\n` 之后的内容
  解决方案：每行一个 text_run，行间插入 `content="\n"` 的 text_run
- **normalize_block_content()**: 防单字拆分，处理 str/list/dict/None 四种输入
  - 字符串：原样返回 strip()
  - 列表："\n".join 拼接各元素文本
  - 字典：取 text/content/answer/value 字段
  - None：返回空字符串

### 报告生成逻辑
- 对话型 PASS：文本标签（"测试操作："）+ 原生代码块（question_text）+ 文本标签（"测试结果："）+ 原生代码块（answer_text）
- 失败不静默回退，逐智能体隔离
- 最终验证新增：对话 PASS 检查 ≥2 个 block_type=14
