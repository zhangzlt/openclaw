## Agent Market 巡检修复（重点）

### 问题演进（四轮修复）

**第1轮 - click 被 overlay 遮挡**（agent #119）
- 错误: `click [contenteditable] covered by div.flex.gap-2`
- 修复: `self.click()` → `self.focus()` 回退
- 结果: ❌ focus 导致编辑器未初始化光标，insertText 失败 → "消息未进入聊天输入框"

**第2轮 - focus 不初始化编辑器**
- 根因: Playwright `focus()` 不设置光标/选区，`insertText` 无法定位
- 修复: `focus()` → `click()` with JS `element.click()` 回退（绕过 hit-test 但保留完整初始化）
- 结果: ❌ 仍有 "消息未进入聊天输入框"

**第3轮 - 搜索框误判**
- 根因: `_detect_chat_input` 用 CSS 选择器，在详情页误把 Search 搜索框当聊天输入框
- 修复: 重写为 JS 检测 + `el.closest()` 排除搜索框/导航栏
- CSS `[attr*=value i]` 标志在 agent-browser 的 Chromium 中可能不被支持
- 结果: ❌ 企业 agent 仍无聊天界面

**第4轮 - Settings 对话框遮挡** ✅ （当前）
- 根因: `?app-settings=archived` URL 参数会打开 Settings/Archived 对话框，遮蔽聊天界面
  - 对话框让 send button click/Enter 事件被拦截
  - 对话框的 iframe 内虽然有针对 agent 的聊天，但 web 层代码无法穿透 iframe
- 修复:
  - 新增 `_close_dialogs()` 方法：JS `dialog.remove()` + UI 备用点击
  - `_ensure_chat_page()` 重写：统一处理对话框关闭、详情页跳转、New Task 点击
- 手动验证: ✅ 关闭对话框后，主界面 contenteditable + Enter 发送成功
