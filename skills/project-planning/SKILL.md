# /project-planning — 项目架构规划

## 触发条件
用户开始新项目、需求分析、技术选型讨论、架构评审。

## 执行流程

### 1. 需求拆解
- 列出核心功能需求
- 识别非功能需求（性能、安全、扩展性）
- 划分优先级（P0/P1/P2）

### 2. 技术选型矩阵
| 维度 | 选项 | 推荐 | 理由 |
|------|------|------|------|
| 语言 | Python/Node.js | 按场景 | ... |
| 框架 | FastAPI/Express | 按需 | ... |
| 数据库 | PostgreSQL/MySQL/MongoDB | 按需 | ... |
| 部署 | Docker/K8s/Vercel | 按规模 | ... |

### 3. 架构设计输出
- 系统架构图（文字描述）
- 模块划分与职责
- 数据流设计
- API 接口规划
- 关键决策记录

### 4. 开发计划
- 里程碑划分（阶段 1/2/3）
- 每个里程碑的交付物
- 预估工作量
- 风险点与应对方案
