# /review — 综合评审

## 触发条件
里程碑完成、发布前评审、项目整体质量评估。

## 评审维度

### 1. 代码层面
- 调用 code-review 技能进行代码级审查
- 架构合规性检查

### 2. 架构层面
- 模块职责是否清晰
- 依赖方向是否正确
- 扩展性评估

### 3. 测试覆盖
- 检查测试文件存在性
- 关键路径是否有测试
- 测试质量抽查

### 4. 文档完整性
- README 是否完整
- API 文档是否更新
- 部署文档是否可用

### 5. 部署就绪
- 环境变量文档化
- 健康检查端点
- 日志和监控配置

### 6. 输出
```
## 综合评审报告

### 各维度评分
- 代码质量: X/10
- 架构设计: X/10
- 测试覆盖: X/10
- 文档完整性: X/10
- 部署就绪: X/10

### 总体评分: X/10
### 阻塞问题: ...
### 改进建议 (按优先级): ...
```
