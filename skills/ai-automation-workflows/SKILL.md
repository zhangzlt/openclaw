# /ai-automation-workflows — AI 自动化工作流

## 触发条件
用户需要自动化 AI 任务、编排工作流、设置触发器、定期执行 AI 任务。

## 执行流程

### 1. 工作流分析
- 理解业务流程的输入/输出/中间步骤
- 判断可自动化环节
- 确定触发方式（定时/事件/手动）

### 2. 编排设计
```
触发条件 → 步骤1 (数据采集) → 步骤2 (AI处理) → 步骤3 (结果输出) → 通知
```

### 3. 实现方式
- **cron 定时任务**: 用 cron 工具创建定时触发
- **WebHook 触发**: 接收外部事件触发
- **链式编排**: 多步骤顺序/并行执行
- **条件判断**: 根据上一步结果决定后续步骤

### 4. 常用模式
- **数据采集管道**: 定时抓取 → AI 分析 → 报告生成
- **监控告警**: 指标检测 → 阈值判断 → 通知
- **内容生成**: 模板 + 数据 → AI 生成 → 发布
- **批量处理**: 队列 → 并发处理 → 汇总

### 5. 容错
- 失败重试机制（默认 3 次）
- 超时保护
- 故障告警
