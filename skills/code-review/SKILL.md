# /code-review — 代码审查

## 触发条件
用户提交代码、PR 审查、代码质量检查请求。

## 审查维度

### 1. 正确性
- 逻辑错误检测
- 边界条件处理
- 异常处理是否完备
- 类型安全

### 2. 安全性
- 注入风险（SQL/命令/Prompt）
- 敏感信息泄露
- 认证授权漏洞
- 输入校验

### 3. 性能
- 不必要的循环/递归
- 内存泄漏风险
- N+1 查询问题
- 阻塞操作

### 4. 可维护性
- 命名规范
- 函数复杂度（圈复杂度过高）
- 代码重复
- 注释质量

### 5. 输出格式
```
## Code Review Report

### 严重问题 🔴 (必须修复)
- [文件:行号] 问题描述 + 修复建议

### 警告 🟡 (建议修复)
- [文件:行号] 问题描述 + 修复建议

### 建议 🟢 (可选优化)
- [文件:行号] 优化建议

### 总体评分: X/10
### 关键发现: ...
```
