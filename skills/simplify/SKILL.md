# /simplify — 代码简化重构

## 触发条件
代码过于复杂、冗余代码、需要重构优化、技术债务清理。

## 简化策略

### 1. 消除冗余
- 删除未使用的变量/导入/函数
- 合并重复代码块
- 提取公共逻辑为函数

### 2. 降低复杂度
- 拆分过长函数（>50 行）
- 减少嵌套层级（>3 层）
- 简化条件表达式
- 用早期返回代替深层 if-else

### 3. 提升可读性
- 变量/函数重命名（更具描述性）
- 添加必要的类型注解
- 使用解构/展开运算符简化
- 提取魔法数字为常量

### 4. 重构原则
- 保持行为不变（不改变功能）
- 每步小改动，确保可回退
- 重构后运行测试验证

### 5. 输出
```
## 简化报告
- 减少行数: X 行
- 消除重复: Y 处
- 提取函数: Z 个
- 变更文件: ...
```
