# /fewer-permission-prompts — 减少权限弹窗

## 触发条件
用户抱怨权限弹窗太多、频繁被打断、自动化流程受阻。

## 优化策略

### 1. 批量合并操作
- 将多次同类操作合并为一次执行
- 用脚本替代逐条命令
- 批量文件写入合并为单次操作

### 2. 风险分级
- **低风险**: 直接执行（读文件、搜索、运行测试）
- **中风险**: 首次询问，后续同会话自动允许（写文件、安装依赖）
- **高风险**: 始终确认（外部 API 调用、删除操作、推送代码）

### 3. 上下文感知
- 同一任务链中的操作不重复询问
- 用户已确认的目标范围自动放行
- 白名单路径的操作自动执行

### 4. 执行原则
- 在安全前提下减少打断
- 批量操作优于逐条操作
- 用 Makefile/脚本封装高风险操作
- 工作区内操作尽量自动执行

### 5. 输出
- 当前操作的权限级别评估
- 可合并/优化的操作建议
