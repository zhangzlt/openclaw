# /update-config — 配置管理

## 触发条件
用户请求修改配置、添加环境变量、设置 hooks、调整权限。

## 执行流程

### 1. 识别配置类型
- **环境变量** (.env): API keys, 数据库连接, 服务地址
- **Hook 配置**: Webhook URL, 回调函数
- **权限配置**: API 白名单, CORS, 访问控制
- **应用配置**: config.json/yaml/toml

### 2. 操作步骤
1. 先读取现有配置文件
2. 合并/更新而非覆盖
3. 自动生成 .env.example（脱敏版本）
4. 检查配置一致性（引用完整性）

### 3. 安全规则
- 禁止将密钥写入非加密文件
- 自动更新 .gitignore 排除敏感文件
- 环境变量变更时同步更新 .env.example
- 提示用户确认敏感配置变更

### 4. 输出
- 变更摘要
- 受影响的文件列表
- 需要重启的服务列表
