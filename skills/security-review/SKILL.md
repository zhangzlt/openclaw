# /security-review — 安全审计

## 触发条件
发布前安全检查、敏感数据处理、权限变更、合规要求。

## 审计清单

### 1. 密钥与凭证
- 扫描硬编码密钥（API Key/Token/密码）
- 检查 .env 是否在 .gitignore
- 验证 .env.example 不包含真实密钥

### 2. 依赖安全
- 检查过时依赖（npm audit / pip-audit）
- 已知漏洞 CVE 扫描
- 依赖供应链风险

### 3. 输入安全
- SQL 注入风险
- XSS 风险
- Prompt 注入风险
- 文件上传安全

### 4. 认证授权
- JWT 过期设置
- CORS 配置审查
- 接口权限校验

### 5. 数据保护
- 敏感数据加密存储
- 日志中的脱敏
- 数据传输加密

### 6. 输出
```
## 安全审计报告
- 风险等级: 低/中/高/严重
- 发现漏洞: N 个
  - [严重] ... (修复建议)
  - [高危] ...
  - [中危] ...
  - [低危] ...
- 合规建议: ...
```
